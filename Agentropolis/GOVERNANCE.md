# Project Governance

## Decision Making

**Now (Phase 0-1)**: Maintainer-led with community feedback via GitHub Discussions.

**Phase 2 (100-1000 stars)**: Core team of 3-5 trusted contributors; decisions by consensus.

**Phase 3 (1000+ stars)**: RFC process (min 2 weeks) + core team voting.

## Roles

**Maintainer**: Merges PRs, releases, security, sets direction.
**Core Contributor**: Regular quality contributions; can merge PRs; private comms access.
**Contributor**: Anyone with merged PR; can review informally.
**Community Member**: Any user; issues/discussions welcome.

## Becoming Core Contributor

Requirements:
- 5+ PRs merged over 3 months
- Good judgment, alignment with values
- Positive community interactions
- Mentorship of newcomers

Expectations: review PRs, attend planning, triage issues, professional representation.

## Conflict Resolution

1. Evidence-based discussion
2. Prototype/data to decide
3. External advice (community vote, experts)
4. Maintainer/core team vote (last resort)

## Transparency

- Decisions documented in minutes/
- Roadmap public
- Security post-mortems published (redacted if needed)

## Financial Sustainability

If funding: infrastructure costs, bug bounties, contributor sprints, paid maintainer time. Decisions by maintainer + core team consensus.
